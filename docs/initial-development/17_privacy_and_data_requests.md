
# 個人情報・データ請求対応

## 方針

MVPでは、個人情報の開示・訂正・削除・利用停止請求は問い合わせベースで対応する。

ユーザー向けの自動エクスポート機能・自動削除機能・自動匿名化機能はMVP対象外とする。

ただし、将来対応できるように、個人情報を含む可能性があるデータ所在と運用手順を明文化する。

## 前提

主な対象は日本国内ユーザーを想定する。

個人情報保護法における保有個人データの開示・訂正・利用停止・消去などの請求に対応できる運用を用意する。

GDPR等の海外規制については、MVPでは専用機能を作らないが、考え方として本人によるアクセス、訂正、削除等の請求を問い合わせベースで扱えるようにする。

## MVPで作らないもの

- ユーザー向けデータエクスポート画面
- ユーザー向け完全削除画面
- 自動匿名化ジョブ
- 365日経過データの自動削除ジョブ
- GDPR専用管理画面
- データ請求管理ワークフロー画面

## データ所在

個人情報または個人に紐づく情報を含む可能性があるテーブル。

| Table | 含まれる可能性がある情報 |
|---|---|
| users | email, display_name, slug, avatar_image_path, bio_markdown |
| social_accounts | provider_user_id, provider_email, provider_nickname, avatar_url |
| user_external_links | URL, label |
| community_admins | 管理者ユーザーID |
| event_admins | イベント個別管理者ユーザーID |
| event_applications | 申し込み状態、参加履歴、キャンセル情報、no-show情報 |
| questionnaire_answers | アンケート回答 |
| payments | 決済ログ、支払いユーザーID、金額 |
| audit_logs | 操作ユーザーID、IPアドレス、User-Agent、操作履歴 |
| stripe_webhook_events | Stripe Webhook payload。個人情報を含む可能性あり |

## 退会と個人情報削除の違い

ユーザー退会はSoft Deleteであり、個人情報の完全削除ではない。

退会済みユーザーは公開画面・参加履歴画面などで「退会済ユーザ」と表示する。

退会後も、以下は運営上保持される可能性がある。

- 申し込み履歴
- 決済ログ
- 監査ログ
- チェックイン情報
- no-show情報
- アンケート回答

個人情報の削除・匿名化は、問い合わせベースで個別対応する。

## 開示請求対応

MVPでは、ユーザーから個人情報開示依頼があった場合、サービス運営者が手動で対応する。

### 手順

1. 問い合わせ窓口で依頼を受け付ける
2. 本人確認を行う
3. 対象ユーザーに紐づくデータを調査する
4. 開示可能なデータを抽出する
5. 安全な方法で本人に提供する
6. 対応内容を監査用に記録する

## 訂正請求対応

MVPでは、ユーザー自身が変更できるプロフィール情報は画面から変更する。

画面から変更できない情報について訂正依頼があった場合は、問い合わせベースで運用対応する。

## 削除・利用停止請求対応

MVPでは、自動削除機能を作らない。

削除・利用停止依頼があった場合は、以下を考慮して運用対応する。

- 法令上、保持が必要な可能性がある情報
- 決済・返金・問い合わせ対応に必要な情報
- 監査ログとして必要な情報
- コミュニティ運営上の安全確保に必要な情報

削除が難しい場合は、可能な範囲で匿名化・非表示化を検討する。

## Stripe Webhook Payload

`stripe_webhook_events.payload` は個人情報を含む可能性がある。

MVPではWebhook冪等性・障害調査を優先しpayloadを保存する。

ただし、365日保持方針の対象とし、Post-MVPでは以下を検討する。

- payloadのマスキング
- 必要最小限の項目のみ保存
- 365日経過後の削除
- 再処理に必要な情報のみ保持

## データ保持期間

主要業務データの保持期間は365日とする。

MVPでは自動削除・匿名化ジョブは実装しない。

Post-MVPで以下を検討する。

- 365日経過データ削除ジョブ
- アンケート回答削除ジョブ
- Stripe Webhook payload削除またはマスキング
- ユーザー退会後の匿名化ジョブ
- ユーザーデータエクスポート機能

## 本人確認

MVPでは本人確認手順を運用で定める。

例:

- 登録メールアドレスからの問い合わせ
- ログイン後の問い合わせ導線
- 必要に応じた追加確認

## 設計上の注意

AI/Codexは以下を勝手に実装しない。

- 自動完全削除
- GDPR専用画面
- データエクスポート画面
- Webhook payloadの保存停止
- 個人情報のログ出力
- 退会時の関連データ一括削除
