# テスト戦略

## 方針

- Pest
- Feature Test中心
- 重要ドメインロジックのみUnit Test
- E2E/VRTはMVP対象外

## Feature Test対象

登録/メール確認/未確認ログイン拒否、Google/GitHubログイン、コミュニティ作成/管理者追加/削除ブロック、イベント作成/公開/自動公開/中止/draft戻し/archive、イベント個別管理者、参加枠作成、有料枠作成時のStripe連携必須、有料枠キャンセル待ち禁止、無料/有料申し込み、Stripe Checkout作成、Stripe Webhook決済成功、支払い期限切れ、抽選/強制当選/強制落選、無料枠キャンセル待ち/自動繰り上げ、参加者キャンセル/管理者キャンセル、事前アンケート、CSVダウンロード、手動/QRチェックイン、二重チェックイン、no-show、権限チェック。

## Unit Test対象

申し込み状態遷移、is_active整合、定員消費判定、キャンセル可能期限判定、抽選処理、QR HMAC生成/検証、slug validation、イベント公開状態判定、有料枠作成条件。

## CI

Laravel Pint、Pest、npm build、Docker build。

## 追加テスト方針

### 用語

ユーザー向け画面では「コミュニティ運営者」表記を優先する。

### 有料枠

- 有料枠で抽選制を選べないこと
- 有料枠の支払い期限が30分になること
- 有料枠ではキャンセル待ちが利用できないこと

### ログイン後導線

- 通常ログイン後 `/dashboard` へ遷移すること
- 申し込み途中ログイン後、元の申し込み画面へ戻ること
- 管理画面要求ログイン後、元の管理画面へ戻ること
