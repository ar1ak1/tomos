# プロジェクト概要

## 一言でいうと

ITエンジニア向けの勉強会申し込みプラットフォーム。参考サービスは connpass / Peatix / TECH PLAY。

## 想定ユーザー

- 参加者: ITエンジニアおよび周辺職。`users` は参加者アカウントを表す。
- コミュニティ管理者: 参加者ユーザーにコミュニティ単位の管理権限が付与された状態。権限は `admin` 1種類のみ。
- イベント個別管理者: 特定イベントのみ全管理権限を持つユーザー。

## MVPで作るもの

- 認証: Laravel Breeze、メール確認必須、Google/GitHubログイン。
- ユーザープロフィール: slug、display_name、正方形アイコン、bio markdown、外部URL。
- コミュニティ: slug、OGP、CoC URL必須、問い合わせメール必須、キャンセルポリシー、特商法等URL、外部URL、Markdown紹介文、管理者一覧公開。
- イベント: Markdown概要、OGP、公開予約、開催日時、都道府県、offline/online/hybrid、会場情報、参加者限定情報、個別管理者。
- 参加枠: 複数枠、先着/抽選、定員、金額、招待枠、枠別募集期間、枠別キャンセル期限、無料枠キャンセル待ち。
- 申し込み: 同意必須、事前アンケート、参加予定・参加履歴、キャンセル。
- 決済: Stripe Connect Accounts v2 API、コミュニティ単位、Stripe Checkout、Webhook、JPY限定。
- チェックイン: 手動チェックイン、QRチェックイン、印刷チケット、no-showマーク。
- 通知: 参加者向け重要メール、管理者向けイベント状態変更メール、サービス運営者向け監視。
- 運用: AWS ECS Fargate、RDS PostgreSQL、S3、CloudFront、SES、EventBridge Scheduler、OpenTofu、GitHub Actions。

## MVPで作らないもの

問い合わせ管理、事後アンケート、分析ダッシュボード、ブラックリスト、外部API連携、システム管理画面、検索、タグ、OpenSearch、有料枠キャンセル待ち、返金機能、領収書/請求書/インボイス発行、通知設定、E2E/VRT、自動削除/匿名化。

## 優先順位

1. コスト
2. 拡張性
3. 速度
4. セキュリティ
5. 安定性

## 想定最大規模

| 項目 | 想定 |
|---|---:|
| ユーザー数 | 100,000 |
| コミュニティ数 | 1,000 |
| 勉強会数 | 150件/日 |
| アクセス数 | 100,000件/日 |



## サービス名

- Service Name: `Tomos`
- Domain: `tomos.dev`

Tomos は「灯す」を由来とするサービス名であり、コミュニティの場や知識の循環に火を灯す、という思想を表す。

## UI / トーン

MVPでは、Tomos のブランドに合わせてアンバー基調 + オレンジ差し色を採用する。

- Primary: `amber-600` / `#D97706`
- Accent: `orange-500` / `#F97316`
- Background: `stone-50` / `#FAFAF9`
- Text: `slate-900` / `#0F172A`

UIトーンは以下を重視する。

- シンプル
- 安心感
- 読みやすい
- 運営しやすい
- 少しあたたかい
- エンジニア向けに過剰装飾しない

ダークモードはMVP対象外とし、ライトモード固定で実装する。

詳細は `16_ui_guidelines.md` を参照する。
