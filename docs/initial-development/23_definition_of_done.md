
# Definition of Done

## 方針

Tomosでは、実装タスクを「動いた」だけで完了にしない。

仕様、テスト、認可、セキュリティ、ドキュメント更新を確認して完了とする。

## 共通Done条件

すべての実装タスクは、原則として以下を満たす。

- 仕様書に沿っている
- MVP対象外機能を追加していない
- 関連するFeature TestまたはUnit Testがある
- Laravel Pintが通る
- Pestが通る
- npm buildが通る
- DB変更がある場合、migrationがある
- DB制約が必要な仕様はmigrationに反映されている
- 権限チェックが必要な画面・操作にはPolicy/Gate等がある
- 個人情報をログに出していない
- 決済・認証・認可・個人情報まわりを簡略化していない
- 仕様変更が発生した場合、docsが更新されている
- 既存仕様と矛盾していない

## UI実装のDone条件

UIを含むタスクは以下も満たす。

- `docs/16_ui_guidelines.md` に沿っている
- Tomosのブランドカラーを使っている
- ユーザー向けUIでは「コミュニティ運営者」表記を優先している
- Danger操作には確認画面または確認導線がある
- 色だけで状態を伝えていない
- フォームエラーが入力欄の近くに表示される
- ダークモードを勝手に実装していない

## 認証・認可タスクのDone条件

認証・認可を含むタスクは以下も満たす。

- 未ログイン時の挙動が明確
- 権限なしの場合の挙動が明確
- メール未確認ユーザーがログインできない
- ログイン後のリダイレクトが仕様通り
- Feature Testがある

## 決済タスクのDone条件

決済を含むタスクは以下も満たす。

- Stripe Webhook署名検証がある
- Webhook処理に冪等性がある
- クレジットカード番号を保存していない
- プラットフォーム手数料を設定していない
- 返金機能を勝手に作っていない
- 本システム上の売上とStripe実売上がずれる可能性を仕様通り扱っている

## メールタスクのDone条件

メール送信を含むタスクは以下も満たす。

- `docs/19_email_guidelines.md` に沿っている
- 件名形式が `[Tomos] {通知内容} - {イベント名}` になっている
- 文体が「です・ます」
- 参加者向けメールに `contact_email` が含まれる
- 有料イベントメールにキャンセルポリシーと特商法URLが含まれる
- 参加者限定情報を不用意にメール本文へ載せていない
- 送信失敗が運用上検知できる

## CSVタスクのDone条件

CSVを含むタスクは以下も満たす。

- `docs/20_csv_and_seed.md` に沿っている
- CSVダウンロードが監査ログ対象になっている
- 個人情報を含むCSVの権限チェックがある

## Docs更新条件

以下の場合はdocs更新を必須とする。

- DBスキーマ変更
- 状態遷移変更
- 認可ルール変更
- URL変更
- メール文面ルール変更
- Stripe処理変更
- 個人情報の所在追加
- MVP対象範囲変更
- UI用語変更

## 完了報告フォーマット

対話式開発では、作業完了時に以下を報告する。

```md
## Summary

## Changed Files

## Tests

## Docs Updated

## Remaining Issues
```
